Memory Networks及其变体

2017-09-29 | note ， paper-reading | |
3,133

Do you remember?

概述

QA问题中求解模型往往是与Memory Networks相关。本次论文笔记主要顺着该模型的演变进行，在介绍完模型的框架之后，分析其不足及（可能的）解决方案。限于篇幅，每个模型应用的任务及实验结果都未予介绍。

Memory Networks

本文是Memory Networks系列的奠基之作。该文着眼于解决长距离情况下的知识利用问题：当前主流解决长距离依赖的RNN模型中虽然有“内存”（即模型的隐状态与权重），但因为实在太小而不能很好的应用在QA情形下。

一般框架

作者们为了解决该问题，提出了Memory Networks这一通用框架，记为MemNN：

如上图所示，MemNN除了组件memory外，还含有四个主要组件：

  • Input : 将输入转换到内部特征空间
  • Generalization : 根据输入更新内存。
  • Output : 根据输入与当前内存状态得到输出（位于特征空间），或者说是有助于输出正确答案的“支持内容”
  • Response : 将Output输出的值转换为需要的格式，如自然语言。

形式化的说，假定输入为$$x$$，则模型处理的流程如下：

  1. 将$$x$$转化为特征表示$$\mathrm{I}(x)$$
  2. 根据新的输入更新memory $$\mathbf{m}_i$$：$$\mathbf{m}_i = \mathrm{G}(\mathbf{m}_i, \mathrm{I}(x), \mathbf{m}), \forall i$$
  3. 根据新的输入与内存状态计算输出的特征$$o$$：$$o = \mathrm{O}(\mathrm{I}(x), \mathbf{m})$$
  4. 将输出状态$$o$$解码为最终的响应：$$r = \mathrm{R}(o)$$

这一框架的提出贡献可能是有效的将QA问题分解为两步：先根据给定的问题找到最相关的内存，再根据这些信息产生答案。（注：之所以说是可能，是因为并没有太关注该文发出之前QA的解决方案。该结论来自于文[2]。）

基本实现

作者为了验证这一框架的可行性，使用神经网络实现了上述的框架并应用于多个任务上。限于篇幅，这里从简介绍。

  1. Input : 将文档与问题转换为稠密表示
  2. Generation : 此处只是简单的将组件input的输出存放到内存的空余位置上
  3. Output : 假定需要找出的支持内容有$$k=2$$个时，
    $$o_1 = \mathrm{O}_1(x, \mathbf{m}) = \underset{i=1, \cdots, N}{\mathrm{argmax}}\ \mathrm{s}_O(x, \mathbf{m}_1)$$
    $$o_2 = \mathrm{O}_2(x, \mathbf{m}) = \underset{i=1, \cdots, N}{\mathrm{argmax}}\ \mathrm{s}O([x, \mathbf{m}{o_1}], \mathbf{m}_i)$$
    其中，$$\mathrm{s}_O$$是关于句子$$x$$与内存$$\mathbf{m}_i$$的匹配打分函数。
  4. Response : 经过上述处理，输入到组件response的就会是$$[x, \mathbf{m}_{o_1}, \mathbf{m}_{o_2}]$$。该组件将根据这些信息选择最合适的单词作为答案（限制题型的正确答案为单词）：
    $$r = \underset{w\in\mathbf{W}}{\mathrm{argmax}}\ \mathrm{s}R([x, \mathbf{m}{o_1}, \mathbf{m}_{o_2}], w)$$
    其中$$W$$是单词表中的所有单词，$$\mathrm{s}_R$$为匹配打分函数。

还有一个关键就是目标函数的制定，对于上述任务的优化函数如下：
$$\sum_{\bar{f}\neq\ \mathbf{m}_{o_1}}\ \mathrm{max}(0, \gamma - \mathrm{s}O(x, \mathbf{m}{o_1}) + \mathrm{s}O(x, \bar{f}))\ +$$
$$\sum{\bar{f}^{‘}\neq\ \mathbf{m}_{o_2}}\ \mathrm{max}(0, \gamma - \mathrm{s}O([x, \mathbf{m}{o_1}], \mathbf{m}_{o_2}) + \mathrm{s}O([x, \mathbf{m}{o_1}], \bar{f}^{‘}))\ +$$
$$\sum_{\bar{r}\neq\ r}\ \mathrm{max}(0, \gamma - \mathrm{s}R([x, \mathbf{m}{o_1}, \mathbf{m}_{o_2}], r) + \mathrm{s}R([x, \mathbf{m}{o_1}, \mathbf{m}_{o_2}], \bar{r}))$$
其中，$$x$$为问题输入，$$r$$、$$\mathbf{m}_{o_1}$$、$$\mathbf{m}_{o_2}$$为正确的响应信息与支持语句，而$$\bar{f}$$、$$\bar{f}^{‘}$$和$$\bar{r}$$都是随机抽取的负样例，$$\gamma$$则是正负样例的间隔。

基本实现的扩展

作者还在基本实现下对更多情形下的情况进行了扩展：

  1. 输入不是完整的句子而是字符时，可以通过在组件input中加入“分块”功能实现
  2. 当组件memory将存放海量数据时，如上的操作（涉及到$$\mathrm{argmax}$$）性能将大幅下降，此时可以考虑哈希或聚类的方法来减少计算量
  3. 组件memory内存储的内容时效性并不相同，对于某些任务可能最近的信息最有用，为此可以将时间信息融入到函数$$\mathrm{s}_{O_t}$$中
  4. 处理自然语言问题时，未登录词也是需要关注的：可以通过上下文信息来估计未登录词的含义
  5. 低维稠密的向量表示有个缺陷就是无法有效利用词语的匹配信息，而这可以通过设计适当的匹配打分函数来改善

我的理解

模型可改进的地方有：

  1. 观察模型的目标函数可以看到我们需要两部分的标记数据：答案与支持语句，而在很多问答任务上很难获取到支持语句。
  2. 在Output查找支持内容时需要指定$$k$$的取值，而支持语句的多少往往与问题有关，是否可以让模型自动选取一个合适的$$k$$呢？

可能的解决方案：
关于问题一，
改动一：去除目标函数中的关于 支持语句 的评分项。但这样去除之后很可能导致模型性能受制于组件output挑选的“支持内容”，最终导致模型训练困难或者训练缓慢。为此采用改动二。
改动二：将组件output的计算函数由$$\mathrm{argmax}$$换成类似attention机制的$$\mathrm{softmax}$$，输出关于各个内存槽的权重；而组件response在计算输出时除了接收到该权值外，还可以利用读取组件memory的内容。这样做的代价可能是组件response的计算性能下降。

关于问题二，
改动一：借鉴RNN的思路，是否可以让组件output循环的选择支持内容，但停止的标准呢？直观的选择是人工指定一个上限，达到该上限时停止循环。但更有效的方法可能是改动二的内容。
改动二：将组件response挑选出的答案的 置信度 （如匹配打分函数输出的分数）发送给组件output，达到一定阈值之后停止循环。但这个过程中又引入了一个超参，是否有方法让这个阈值也通过模型自动学习呢？值得思考。

End-to-End Memory Networks

本文的题目明确的表达了工作致力解决的问题：让MemNN可以端到端的训练。下文中将这样的模型记为MemN2N。

模型

该文的主要贡献就是提出了方案去除了MemNN需要大量标记数据的弊端；除此之外，还借鉴了RNNsearch的方法，对于需要多步推导的问题使用多个计算步骤进行解决。作者将其中的每个计算步骤称为hop。
模型整体框架如下图，(a)是单次计算过程的示意图，即只有一个hop；(b)是有多次计算步骤的示意图。

简单的说，模型要处理的问题就是：读入多个输入 $$\mathbf{x}_1, \mathbf{x}_2, \cdots, \mathbf{x}_n$$，转化为稠密向量之后存入到组件memory中；之后读入问题 $$\mathbf{q}$$，经过计算后输出答案 $$\mathbf{a}$$。其中，$$\mathbf{x}_i, \mathbf{q}, \mathbf{a}$$ 的词都来自于一个大小为 $$\vert V\vert$$ 的词表。

单层结构

与MemNN中类似，MemN2N也有组件memory，但去除了组件Generalization：

  • Input Memory Representation : 如图(a)所示，句子 $$\mathbf{x}_i$$ 在向量化之后与矩阵 $$\mathbf{A}, \mathbf{C}$$ 相乘得到表示 $$\mathbf{m}_i, \mathbf{c}_i$$，同样向量化之后的问题 $$\mathbf{q}$$ 与矩阵 $$\mathbf{B}$$ 相乘得到表示 $$\mathbf{u}$$。其中，表示 $$\mathbf{c}_i$$ 输入到组件output中，表示 $$\mathbf{m}_i$$ 再与 $$\mathbf{u}$$ 经过下述计算得到权重再输入到组件output中：
    $$\mathbf{p} = \mathrm{softmax}(\mathbf{u}^T\ \mathbf{m})$$
  • Output Memory Representation : 该处类似于attention机制，通过计算 $$\mathbf{p}, \mathbf{c}$$ 得到 $$\mathbf{o}$$：
    $$\mathbf{o} = \sum_{i}\ p_i\ \mathbf{c}_i$$
  • Generating the Final Prediction : 如图所示，将通过如下公式选出单词：
    $$\hat{a} = \mathrm{softmax}(\mathbf{W}\ (\mathbf{o}\ +\ \mathbf{u}))$$

之后就可以根据真实答案与输出答案计算交叉熵，利用误差反向传播算法优化模型。

注：MemN2N与MemNN并不完全对应。

多层结构

如图(b)所示，多层memory通过以下方式堆叠在一起：

  • 除第一层外，其他层的输入 $$\mathbf{u}^{k+1} = \mathbf{u}^{k}\ +\ \mathbf{o}^{k}$$
  • 每一层都有自己独立的矩阵 $$\mathbf{A}, \mathbf{C}$$，下文会接着介绍如何共享参数
  • 最后一层产生答案的方式为 $$\hat{a} = \mathrm{softmax}(\mathbf{W}(\mathbf{o}^k\ +\ \mathbf{u}^k))$$

作者提出了两种共享参数的方式：

  1. Adjacent :
    (a) $$\mathbf{A}^{k+1} = \mathbf{C}^k$$
    (b) $$\mathbf{W}^T = \mathbf{C}^k$$
    (c) $$\mathbf{B} = \mathbf{A}^1$$
  2. Layer-wise :
    (a) $$\mathbf{A}^1 = \mathbf{A}^2 = \cdots = \mathbf{A}^k$$
    (b) $$\mathbf{C}^1 = \mathbf{C}^2 = \cdots = \mathbf{C}^k$$
    (c) 添加一个矩阵 $$\mathbf{H}$$ 用于更新hops之间的 $$\mathbf{u}$$，即 $$\mathbf{u}^{k+1} = \mathbf{H}\mathbf{u}^k\ +\ \mathbf{o}^k$$

我的理解

MemN2N的整体结构与MemNN对应关系并不十分明确。若是按照MemNN的划分，组件memory的功能更多的由在多个hop间传递的矩阵 $$\mathbf{A}, \mathbf{C}$$ 来担当；组件input隐含到将文档与问题转换为稠密表示的过程中，而这个过程使用的是词袋模型，会丢失顺序信息；没有功能对应组件generation；组件output与组件response基本与 Output Memory Representaion 和 Generating the Final Predication 相对应。另外，作者在文中提到使用Layer-wise进行参数共享时，该模型就类似于RNN模型。但是区别在于该模型更新隐层状态 $$\mathbf{u}_k$$ 时，会根据文档进行更有针对的更新（描述不全面，待补充）。

模型可改进的地方有：

  1. 如上分析，该模型实质上使用的是 词袋模型，会丧失语序信息
  2. 需要手动指定hops的数量

可能的解决方案：
关于问题一，
方案一：使用RNN类的模型对句子进行建模，但是可能会过度压缩语义信息。为此可以采用方法是保存词语层面的稠密向量表示来取代矩阵 $$\mathbf{A}, \mathbf{C}$$，但这种方案只限于文档的句子数量不多的情形。
方案二：针对方案一的局限，可以考虑先筛选出无关的句子，再花费资源到词语层面进行计算。例如，使用RNN类模型得出句子向量与问题对应句子向量计算，若不相关则不进行词语层面的分析，直接进入下一个hop。

关于问题二，
方案：与MemNN类似的方案类似，并且由于MemN2N的端到端特性更容易进行，即根据最终输出词语的 置信度 选择是否要停止，并制定相关阈值与hops的上限值。

作者其实针对问题一，也提出了解决方案，就是在将文档与问题编码为稠密向量时也考虑 词语在句中位置 的影响。另外，他们还针对输入文档时缺少 时序 信息提出了类似的解决方案，在编码稠密向量的时候考虑 时序 的影响。（此处略去公式，但依旧存疑的是为什么选用这样的公式？）
