Blocks学习手册（三） - bricks

2017-03-27 | howto ， theano ， blocks | |
1,455

Blocks is a framework that helps you build and manage neural network models on using Theano.

概述

在Blocks学习手册（一） - 概览 中我们说到了构建神经网络的基本单元bricks。我们现在就来详细了解下这种对象的作用与生命周期，以便我们更好的使用该类。

生命周期

bricks实质上是parametrized Theano operations。brick通常又一些attributes和parameters定义组成。其中，attributes是用来定义Block的（如，输入向量的维数等）；后者parameters则是指模型训练过程中不断计算的参数（如，W和b）

brick的生命周期涉及到如下四个方面：

  1. Configuration：设置（全部或者部分）brick的attributes。当我们通过构造函数常见brick并传入一些值时，或者我们改变brick对象的属性时，都算在进行configuration。这个步骤中不会有theano变量被创建。
  2. Allocation：（可选）创建brick对象代表的theano shared variables。当调用allocate()时，theano变量将被创建并被默认初始化为NaN。
  3. Application：这一步是为了构建出Theano中的计算图（Computational graph），根据attributes将输入与输出（parameters）连接起来。这一步只有当对象被完全配置好后才可以进行。
  4. Initialization：设置brick中parameters，即theano变量的实际数值。用户蹄冻的量将替换掉默认的初始化值。

注意：当brick对象没有调用allocate()而直接调用apply()方法时，将静默调用allocate()方法。

nested bricks

许多神经网络模型，特别是那些复杂模型都可以当成层次模型。即便是简单如MLP，也可以分为线性变换层与非线性变换层。

具体到Blocks上，就是说bricks也可以有children。parent bricks负责配置children bricks，即保证children bricks的配置是合法的。

通过上例可以看出MLP自动构造了child bricks进行线性变换。当我们将MLP应用到x上时，它自动的配置好了child bricks的输入维数与输出维数。相同的，当我们调用initialize()时，它自动的初始化了child bricks的参数。
