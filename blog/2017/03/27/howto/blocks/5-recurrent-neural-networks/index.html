Blocks is a framework that helps you build and manage neural network models on using Theano.

概述

该部分解释Blocks中的recurrent bricks。如果读者不熟悉Blocks中的bricks概念的话，请阅读Blocks学习手册（三） - bricks之后再食用此文。

简单示例


我们先实现上图所述的简单recurrent bricks，即累加它所有的输入。上图数学表示就是：
$$
\mathbf{h}t = \mathbf{h}{t-1} + \mathbf{x}_t
$$

我们再来看看下面这个复杂一点的recurrent bricks，它将输入乘上2倍之后再累加：

数学表达式为：
$$
\mathbf{h}t = \mathbf{h}{t-1} + 2 \cdot \mathbf{x}_t
$$

在上例中，我们可以看到为了让输入乘上2。我们对x应用了线性变换bricks.Linear。实际中的RNN用的表达式为
$$
\mathbf{h}t = \mathsf{f}(\mathbf{V}\mathbf{h}{t-1} + \mathbf{W}\mathbf{x}_t + b)
$$
通过这样涉及recurrent bricks，我们可以更加模块化地，更加灵活地使用该计算单元。例如，$\mathbf{W}\mathbf{x}_t$甚至可以用一个神经网络代替。

初始状态


recurrent model一般都需要指定初始状态的。但是，我们上例中都忽略了recurrent brick的初始状态$\mathbf{h}_0$。其实，这是因为Blocks默认将初始状态设置为0。但是我们其实可以将他们显式设置出来的。如上图所示，我们就是将初始值设置为1：
$$
\mathbf{h}t = \mathbf{h}{t-1} + \mathbf{x}_t, \mathbf{h}_0 = 1
$$

Iterate (or not)

recurrent bricks的apply方法有一个参数为iterate，默认设置为True。这是因为我们提供给这个tensor的输入的维度会比recurrent.SimpleRecurrent.apply()需要的大。多出来的第一个就是对应了我们要迭代的序列长度。
将iterate设置为False可以让方法apply指计算序列中的一步。
上述这些机制让我们将多个recurrent layer结合起来变得相当简单。

这个示例中涉及的东西就很多了。

我们定义了一个名字为FeedbackRNN的recurrent brick，它有两个类型为bricks.recurrent.SimpleRecurrent的child bricks。
这个类中的get_dim是用来输出调用apply方法时所需要的输入维数。
这个类的核心功能在于方法apply。装饰器@recurrent用来指定该方法在那个序列上迭代，该返回什么值，返回值中的哪些对应隐层状态。它的参数inputs与outputs和装饰器@application的作用如下：

  • outputs，与@application的类似，它定义了方法apply返回的所有结果，包括recurrent outpus.
  • states是outputs中与recurrent outputs对应的子集，这意味着sequences与states的并集和@application的inputs相对应。

注意，我们并没有再调用theano.scan()。这是因为apply的实现负责实现该功能，即读取上一个时刻的状态，并输出当前时刻的状态。剩下的东西其实都被装饰器@recurrent实现了。

通过上例我们就可以看出iterate的重要性，它让我们可以在另一个bricks中组装多个recurrent bricks。

提示：当查看recurrent brick相关的文档时，要时刻牢记它的apply方法的参数是与一次迭代相对应的，即假设iterate = False.
