unittest介绍

unittest supports test automation, sharing of setup and shutdown code for tests, aggregation of tests into collections, and independence of the tests from the reporting framework.

unittest是JUnit的python版本，内嵌在Python中。
跟JUnit一样，unittest涉及四个概念：

  1. test fixture： test case的脚手架，用来构建(set up)测试的环境或者在测试之后做些回收资源等拆除(tear down)测试环境。
  2. test case：测试用例，执行具体测试任务的实体。
  3. test suite：把一些test case和/或test suite打包，方便测试的执行。
  4. test runner：用来运行测试用例并输出统计信息。

概念是概念，耍起来的是unittest中的一个个类：
test case:

  1. TestCase：类如其名，用来新建test case
  2. FunctionTestCase: 用来将遗留的测试代码，集成到unittest测试框架中

具体到脚手架(test fixture)，它是针对测试用例而言，不同的测试用例对应的脚手架(test fixture)创建方式是不同的。
test fixtures(Using TestCase)

  • setUp(): 环境搭建放在这儿
  • tearDown(): 收尾工作放在这儿

需要注意的是TestCase每次只跑一个测试方法，因而脚手架(test fixture)的创建与拆除会在每次测试时都进行。

test suite:

  • TestSuite： suite => 套件。是的没错，这是测试的套件。当suite执行时，包含在它里面的测试用例及测试套件都会被执行。

test runner:

  • TextTestRunner: 默认的测试执行者，顺便提下文档中的两个类：

    A test runner is an object that provides a single method, run(), which accepts a TestCase or TestSuite object as a parameter, and returns a result object. The class TestResult is provided for use as the result object.
