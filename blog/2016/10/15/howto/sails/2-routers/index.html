Sails.js, the web framework of your dreams.

路由

Routes are rules that tell Sails what to do when faced with an incoming request. There are two main types of routes in Sails: custom (or “explicit”) and automatic (or “implicit”).

Sails.js提供了两种形式的路由，一是custom，一是automatic。

默认路由

Sails默认为我们制订了一些路由的映射规则。如果一个URL没法匹配任何自定义的路由，此时默认路由就会做出反应。

Blueprint routes

Blueprints are Sails’ way of quickly generating API routes and actions based on your application design.

该路由为controllers和models提供了全方面的REST的API，方便测试。

blueprint API由两部分组成：blueprint routes和blueprint actions，它为我们提供了默认的RESTful JSON API。举例来说，当我们创建了一个User.js的模型与UserController.js的控制器之后，我们就可以通过/user/create?name=jce来创建一个User，通过/user来查询应用中已有的用户，并且这些行为都是可以配置的。

Blueprint Routes

当Sails应用正式运行（lift），并且开启blueprint功能后，应用就会被默认绑定一些路由。默认情况下，blueprint routes会指向各自对应的blueprint action，并且它们都是高度可定制的。
在sails中，blueprint routes有三种：

  1. RESTful routes: 这种情况下，URL的路径是/:modelIdentity或者/:modelIdentity/:id的形式。至于CRUD的操作，则是通过HTTP的方法决定的。例如：POST /user将新建一个User，DELETE /user/123将删除主键为123的User。在生产环境下，RESTful routes 应当被policies限制，防止一些未授权的操作，破坏系统状态。
  2. Shortcut routes: 这与RESTful routes不同指出在于，CRUD动作是在路径中指出的。如，/user/create?name=joe用来创建用户，/user/update/1?name=mike用来更新主键为1的用户信息，同时这些路由的HTTP请求方式都是GET。在生产环境下，Shortcut routes应当被禁止。
  3. Action routes: 这些是为你Controller的自定义action生成路由。如，你创建了一个FooController.js的Controller，它有一个bar方法，那么只要你的blueprint action routes是开启的，/foo/bar这个路由就会自动创建。与上述两种类型的路由不同，action routes不要求有对应的model层。

至于blueprint routes的配置，可以参见blueprints subsection of the configuration references。

Blueprint Actions

注意，不要把blueprint actions与上述的blueprint action routes混淆。
blueprint actions为你的某些满足条件的controller提供默认支持。这些controller的要求就是要有相同的model，如ParrotController需要一个名为Parrot的Model。更具体的说，如果你创建了一个满足条件的UserController（即需要有名为User的model），你的UserController默认就将有find、create、update、destroy、populate、add和remove方法了。
默认情况下，blueprint RESTful routes与blueprint shortcut routes就是绑定到对应的这些方法上的。我们可以通过覆写（override）特定Controller的方法来改变特定方法的行为，如覆写ParrotController.find方法。同时，如果你想改变应用中所有Controller的blueprint action，只需要在特定的位置创建对应的方法就好，例如：api/blueprints/create.js。

现在（2016年10月17日），Sails提供如下blueprint actions：

  • find
  • findOne
  • create
  • update
  • destroy
  • populate
  • add
  • remove

关于如何启用/禁用以及覆写blueprint action，可以参见Blueprint API reference。
